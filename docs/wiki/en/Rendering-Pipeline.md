## Overview

Quarantine Engine uses the **Vulkan API** directly.  
The rendering loop performs explicit synchronisation, command-buffer recording, and queue submission each frame.

---

## Vulkan Initialisation Sequence

```
VulkanInstance           → create VkInstance, enable validation layers
VulkanLayerAndExtension  → enumerate available layers/extensions
DeviceModule             → pick VkPhysicalDevice, create VkDevice
QueueModule              → acquire graphics / present / compute VkQueues
WindowSurface            → create VkSurfaceKHR via GLFW
SwapChainModule          → create VkSwapchainKHR (double / triple buffered)
ImageViewModule          → VkImageView per swapchain image
DepthBufferModule        → depth VkImage + VkImageView (D32_SFLOAT)
AntiAliasingModule       → MSAA resolve attachment (VkSampleCountFlagBits)
RenderPassModule         → VkRenderPass with colour + depth + resolve attachments
FrameBufferModule        → one VkFramebuffer per swapchain image
CommandPoolModule        → VkCommandPool + VkCommandBuffer per frame
SynchronizationModule    → VkSemaphore (image-available, render-finished) + VkFence
```

## Per-Frame Render Loop

```cpp
// Pseudocode mirroring App::DrawFrame()
vkWaitForFences(inFlightFence);                          // 1. wait previous frame
vkAcquireNextImageKHR(swapchain, imageAvailableSem);     // 2. acquire image

vkResetCommandBuffer(commandBuffer);
vkBeginCommandBuffer(commandBuffer);
  RecordShadowPass(commandBuffer);                       // 3a. depth shadow passes
  RecordMainRenderPass(commandBuffer);                   // 3b. PBR geometry
  RecordComputeDispatches(commandBuffer);                // 3c. skinning / particles
  RecordAtmospherePass(commandBuffer);                   // 3d. sky
  RecordParticlePass(commandBuffer);                     // 3e. transparent particles
  RecordDebugPass(commandBuffer);                        // 3f. debug overlays
  RecordEditorGUI(commandBuffer);                        // 3g. ImGui
vkEndCommandBuffer(commandBuffer);

vkQueueSubmit(graphicsQueue, commandBuffer,              // 4. submit
              waitSem=imageAvailableSem,
              signalSem=renderFinishedSem,
              fence=inFlightFence);

vkQueuePresentKHR(presentQueue, renderFinishedSem);      // 5. present
```

## Render Passes

### Shadow Pass

Two shadow sub-passes run before the main geometry pass:

| Sub-pass | Class | Output |
|---|---|---|
| Cascaded Shadow Map (CSM) | `ShadowPipelineManager` + `CSMResources` | Depth array image (N cascades) |
| Omnidirectional Shadow | `ShadowPipelineManager` + `OmniShadowResources` | Depth cubemap per point light |

Cascade splits follow a logarithmic distribution.  
The main pass samples shadow maps using PCF filtering (via `QEShadows.glsl`).

### Main Geometry Pass

Draws all opaque `QEGeometryComponent` objects with the default PBR pipeline:

- **Vertex shader:** `resources/shaders/Default/default.vert` — transforms vertices, computes TBN matrix.
- **Fragment shader:** `resources/shaders/Default/default.frag` — PBR BRDF, shadow sampling, atmosphere scattering.

### Mesh Shader Pass (optional)

When the `VK_EXT_mesh_shader` extension is available, a task + mesh shader pipeline (`resources/shaders/Mesh/`) can replace the vertex pipeline.  
Meshlets are generated by `meshoptimizer` and stored in `Meshlet` structures.

### Atmosphere Pass

Renders the sky dome using pre-computed look-up tables:

| LUT | Compute Shader |
|---|---|
| Transmittance LUT | `transmittance_LUT.comp` |
| Multi-scattering LUT | `multi_scattering_LUT.comp` |
| Sky-view LUT | `sky_view_LUT.comp` |

The final sky is drawn by `atmosphere.vert` / `atmosphere.frag` or `sky_spherical_map.*`.

### Debug / Editor Passes

| Pass | Shader | Purpose |
|---|---|---|
| AABB wireframe | `debugAABB.vert/frag` | Physics collider outlines |
| Debug lines | `debug.vert/frag` | Bone gizmos, camera frustum |
| Grid | `grid.vert/frag` | Editor ground grid |
| ImGui | Dear ImGui Vulkan backend | Editor UI |

## Graphics Pipelines

`GraphicsPipelineManager` (singleton) owns all `GraphicsPipelineModule` instances.  
Each module wraps a `VkPipeline` + `VkPipelineLayout` + `VkDescriptorSetLayout`.

```cpp
// Retrieve a named pipeline
auto* pipeline = GraphicsPipelineManager::Instance()->Get("default_pbr");

// Bind and draw
pipeline->Bind(commandBuffer);
vkCmdDrawIndexed(commandBuffer, indexCount, 1, 0, 0, 0);
```

### Shader Stages Supported

| Stage | Vulkan Stage Flag |
|---|---|
| Vertex | `VK_SHADER_STAGE_VERTEX_BIT` |
| Fragment | `VK_SHADER_STAGE_FRAGMENT_BIT` |
| Geometry | `VK_SHADER_STAGE_GEOMETRY_BIT` |
| Tessellation Control | `VK_SHADER_STAGE_TESSELLATION_CONTROL_BIT` |
| Tessellation Evaluation | `VK_SHADER_STAGE_TESSELLATION_EVALUATION_BIT` |
| Compute | `VK_SHADER_STAGE_COMPUTE_BIT` |
| Task (EXT) | `VK_SHADER_STAGE_TASK_BIT_EXT` |
| Mesh (EXT) | `VK_SHADER_STAGE_MESH_BIT_EXT` |

## Descriptor Management

Descriptors are managed per-pass:

| Class | Purpose |
|---|---|
| `DescriptorBuffer` | Per-object UBO / sampler descriptors |
| `ComputeDescriptorBuffer` | Compute shader storage buffers |
| `CSMDescriptorsManager` | Cascaded shadow map sampler descriptors |
| `PointShadowDescriptorsManager` | Omnidirectional shadow cubemap descriptors |

## Uniform Buffer Objects

UBO structures are defined in `src/Memory/UBO.h` and updated once per frame:

```cpp
struct ViewProjectionUBO {
    glm::mat4 view;
    glm::mat4 projection;
    glm::vec3 cameraPosition;
};

struct ModelUBO {
    glm::mat4 model;
    glm::mat4 normalMatrix;
};
```

## Multi-Sampling (MSAA)

The engine supports MSAA up to the hardware maximum.  
The sample count is queried from `VkPhysicalDeviceProperties` and configured in `AntiAliasingModule`.  
MSAA resolve is performed as part of the main render pass's resolve attachment.

---

## Ray Tracing

`RayTracingModule` builds **Bottom-Level Acceleration Structures (BLAS)** per mesh and a **Top-Level Acceleration Structure (TLAS)** for the scene.  
The extension `VK_KHR_acceleration_structure` is used.  
Full ray-traced shading is a future work item; the infrastructure is in place for hybrid rendering.
